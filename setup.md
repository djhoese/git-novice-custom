  <h3>Git</h3>
  <p>
    Git is a version control system that lets you track who made changes
    to what when and has options for easily updating a shared or public
    version of your code
    on <a href="https://github.com/">github.com</a>. You can also host it
    on the SSEC GitLab server <a href="https://gitlab.ssec.wisc.edu/">gitlab.ssec.wisc.edu</a>.
    You will need a
    <a href="https://help.github.com/articles/supported-browsers/">supported
    web browser</a>.
  </p>
  <p>
    You will need an account at <a href="https://gitlab.ssec.wisc.edu/">gitlab.ssec.wisc.edu</a>
    for parts of the Git lesson. SSEC employees should already have an account.
    If you can not sign in to GitLab please contact TC <b>before</b> the workshop.
  </p>

  <h3>Text Editor</h3>

  <p>
    When you're writing code, it's nice to have a text editor that is
    optimized for writing code, with features like automatic
    color-coding of key words. The default text editor on macOS and
    Linux is usually set to Vim, which is not famous for being
    intuitive. If you accidentally find yourself stuck in it, hit
    the <kbd>Esc</kbd> key, followed by <kbd>:</kbd>+<kbd>Q</kbd>+<kbd>!</kbd> 
    (colon, lower-case 'q', exclamation mark), then hitting <kbd>Return</kbd> to 
    return to the shell.
  </p>

### Working Directory

We'll do our work in the `Desktop` folder so make sure you change your working directory to it with:

~~~
$ cd
$ cd Desktop
~~~
{: .bash}

### GitLab SSH Keys (Optional)

While working with gitlab.ssec.wisc.edu we will be talking to the site using
a protocol called HTTPS. This usually means that you have to enter your
password every time you upload information to GitLab. For future work you may
want to configure SSH Keys for your system and GitLab account to allow for
password-less access.

You can follow [these instructions](https://gitlab.ssec.wisc.edu/help/ssh/README)
for instructions on how to do this for all platforms. Ask TC for any additional
help.
